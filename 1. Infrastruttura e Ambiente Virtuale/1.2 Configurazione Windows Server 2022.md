# 🛠️ <span style="color:#00bfff">Fase 1: Creazione della VM e Allocazione Risorse</span>

### <span style="color:#9932cc">1. Configurazione Iniziale e Compatibilità Hardware</span>
Durante la creazione guidata della macchina virtuale, è stata selezionata la configurazione **Custom (Advanced)** per garantire il massimo controllo granulare sui parametri hardware e di virtualizzazione.

- **Hardware Compatibility:** Workstation 17.x (o versione più recente disponibile). 

- **Scelta d'Installazione:** *I will install the operating system later*. 
  **Perché questa scelta:** Evita l'attivazione dell'Easy Install di VMware, permettendo un controllo granulare e manuale del processo di setup del sistema operativo da vero amministratore di rete.

### <span style="color:#9932cc">2. Identificazione del Guest OS e Nome del Nodo </span>
Il sistema operativo è stato catalogato secondo gli standard Microsoft aziendali per ottimizzare l'integrazione dei driver di virtualizzazione (_VMware Tools_).

- **Guest Operating System**: Microsoft Windows

- **Version**: Windows Server 2022

- **Firmware Type**: UEFI con Secure Boot garantendo l'utilizzo solo di driver firmati

- **Virtual Machine Name**: WS2022-DC01 (Windows Server 2022 Domain Controller 01)

- **Location**: Salvataggio su disco SSD dedicato ad alte prestazioni!

### <span style="color:#9932cc">3. Dimensionamento delle Risorse Logiche</span>
Le specifiche hardware virtuali sono state bilanciate per garantire la stabilità dei servizi di dominio senza saturare l'hardware del PC host.

- **Processor Configuration**: 1 CPU con 2 Core

- **Memory (RAM)**: 4096MB (4GB)
     - _Nota tecnica:_ 4 GB rappresentano il perfetto compromesso per un Domain Controller in ambiente di test, garantendo fluidità nell'esecuzione di Active Directory e del comparto DNS/DHCP.

### <span style="color:#9932cc">4. Isolamento di Rete (Segmentazione LAN)</span>
Per evitare che il nuovo server interferisca con la rete domestica reale (evitando conflitti di IP o l'intercettazione del traffico da parte di altri dispositivi di casa), la scheda di rete è stata completamente isolata.

- **Network Connection**: Use host-only networking ---> Lan Segment
    _Nota Tecnica:_ _Perché lo facciamo?_ Questo collegherà momentaneamente la macchina a uno switch virtuale privato che comunica solo con il tuo PC fisso e le altre VM, togliendo del tutto l'accesso a Internet.
    
    Al termine del wizard, andremo a rifinire questa impostazione trasformandola in un **LAN Segment** (un segmento di rete ancora più isolato e stagno). Ma per superare questa schermata del wizard senza lasciare la VM collegata alla tua rete di casa (evitando i rischi del NAT o del Bridged), la voce _Host-only_ è il trampolino di lancio perfetto.


![[Screenshot 2026-06-11 113831.png|407]]


### <span style="color:#9932cc">5. Controller Type</span>

- **Select I/O Controller Type:** LSI Logic SAS
    **Significato tecnico:** È il chip virtuale che gestisce il passaggio dei dati tra la scheda madre e i dischi. LSI Logic SAS è lo standard più compatibile in assoluto per i sistemi operativi Windows Server.



![[Screenshot 2026-06-11 143239.png|412]]



### <span style="color:#9932cc">6. Tipo di Disco e Storage</span>
Andremo a configurare queste impostazioni per sfruttare al massimo la velocità dell'SSD fisico. Viene definito uno spazio ideale per le specifiche richieste da Windows Server 2022, impostando il disco in modalità dinamica per fare in modo che la macchina occupi memoria reale sul PC host solo man mano che i dati verranno effettivamente scritti. Infine, l'archiviazione viene configurata su un file singolo per consolidare i blocchi di dati ed evitare colli di bottiglia, aumentando sensibilmente le prestazioni di lettura e scrittura della macchina virtuale.

- **Disk Type**: NVMe

- **Select a Disk:** Create a new virtual disk

- **Disk Capacity**: 60GB

# 🛠️ <span style="color:#00bfff">Fase 2: Installazione OS e primo avvio Windows Server 2022</span>


### <span style="color:#9932cc">1. Configurazione del Tipo di Installazione</span>

All'avvio della ISO e dopo aver superato la selezione della lingua, il setup ha presentato la schermata *"Which type of installation do you want?"*.

* **Scelta d'Installazione:** `Custom: Install Windows only (advanced)`.

* **Perché questa scelta:** Scarta l'opzione *Upgrade* (inutilizzabile su un disco vuoto) e garantisce un'installazione pulita da zero, azzerando qualsiasi configurazione pregressa e offrendo la massima stabilità per un ambiente server aziendale.

![[Screenshot 2026-06-11 155526.png]]



### <span style="color:#9932cc">2. Selezione dell'Edizione del Sistema Operativo ed Ambiente</span>

Nella schermata successiva, l'installer richiede di specificare la licenza d'uso e il tipo di interfaccia per l'amministrazione del server. 

* **Scelta d'Installazione:** `Windows Server 2022 Standard Evaluation (Desktop Experience)`.

* **Perché questa scelta:** La dicitura *Desktop Experience* è fondamentale per abilitare l'interfaccia grafica completa (GUI) e la dashboard del Server Manager. Senza questa specifica, il sistema verrebbe installato in modalità *Core* (operabile solo da riga di comando e PowerShell), mentre il nostro obiettivo di laboratorio richiede la gestione visiva dei ruoli.

![[Screenshot 2026-06-11 155412.png]]



### <span style="color:#9932cc">3. Allocazione e Partizionamento del Disco Virtuale</span>

La procedura si sposta sulla gestione dello storage logico all'interno della schermata *Where do you want to install Windows?*. 

* **Scelta d'Installazione:** Selezione diretta del volume `Drive 0 Unallocated Space` (60 GB) e clic su *Next* senza partizionamento manuale. 

* **Perché questa scelta:** Permette a Windows Server di gestire autonomamente lo spazio, formattando il volume e generando in automatico le partizioni strutturali necessarie (inclusa la partizione EFI per il boot protetto UEFI).

![[Screenshot 2026-06-11 155600.png]]



### <span style="color:#9932cc">4. Completamento del Setup, Driver e Primo Riavvio</span>

Il sistema avvia la copia massiva dei dati e la configurazione hardware di background. 

* **Progressione:** Monitoraggio del caricamento nella schermata *Getting files ready for installation*, seguito dal riavvio automatico della macchina virtuale. 

* **Fase di Boot:** Al riavvio, il sistema esegue la fase silente di *Getting devices ready*, durante la quale il kernel rileva le componenti hardware virtualizzate fornite dall'hypervisor e ne adatta i driver.



### <span style="color:#9932cc">5. Configurazione dell'Account Amministratore Locale</span>

Al termine dei cicli di boot, il server richiede la messa in sicurezza dell'utenza locale nativa nella schermata *Customize Settings*. 

* **Scelta d'Installazione:** Assegnazione di una credenziale personalizzata per l'utente `Administrator`. 

* **Perché questa scelta:** Adempimento dei requisiti severi di complessità imposti dall'OS (combinazione di lettere maiuscole, minuscole, numeri e caratteri speciali) per mitigare i rischi di attacchi brute-force in fase di pre-deployment.



### <span style="color:#9932cc">6. Sblocco e Primo Accesso al Sistema</span>

Raggiunta la schermata di blocco di Windows Server, viene eseguito il primo inserimento credenziali e l'atterraggio sul sistema operativo. 

* **Input di Sblocco:** Utilizzo della combinazione di tasti dedicata `CTRL + ALT + INS`. 

* **Perché questa scelta:** Evita che la combinazione fisica *Ctrl+Alt+Canc* venga intercettata dal sistema operativo host reale della macchina fisica, inviando il comando di sblocco esclusivamente all'interno della VM su VMware Workstation. 

* **Esito:** Accesso eseguito con successo sul Desktop e avvio automatico della dashboard centrale del *Server Manager*.



### <span style="color:#9932cc">7. Installazione e Configurazione di VMware Tools</span>

Subito dopo il primo accesso, per ottimizzare le prestazioni, la risoluzione video e l'integrazione tra l'Hypervisor e la macchina virtuale, viene eseguita l'installazione dei driver guest dedicati.

- **Montaggio dell'ISO:** Da VMware Workstation, menu **VM** -> **Install VMware Tools...**, montando l'unità CD-ROM virtuale all'interno del server.
    
- **Esecuzione Setup:** Apertura dell'unità disco da _Esplora Risorse_ ed esecuzione del file `setup64.exe`, selezionando la modalità di installazione **Completa (Complete)**.
    
- **Perché questa scelta:** Questa modalità installa tutti i driver necessari per l'accelerazione grafica, il trascinamento dei file (Drag and Drop), la gestione energetica avanzata e l'ottimizzazione delle schede di rete virtuali.
    
- **Esito:** Installazione completata con successo e riavvio del server eseguito per applicare definitivamente i cambiamenti.
    

> [!INFO] **Controllo di Stabilità Post-Riavvio** Al riavvio della macchina, verificare la fluidità del puntatore del mouse e che l'icona di VMware Tools sia correttamente attiva nella barra delle applicazioni in basso a destra
